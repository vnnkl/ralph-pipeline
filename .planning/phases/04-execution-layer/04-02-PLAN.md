Replace the stub templates/execute.md with a production template.

The template must implement these steps:

**Step 1: Read Context**
- Read files_to_read (STATE, config)
- Discover beads: use Glob to list .beads/*.md, sort alphabetically
- Count total beads. If zero: PHASE FAILED with "No beads found in .beads/. Run the convert phase first."

**Step 2: Present Execution Gate**
- Use AskUserQuestion with these options:
  1. **Manual** (DEFAULT) -- Launch ralph-tui for interactive execution
  2. **Headless** -- Run claude -p per bead automatically
- Manual is the default option (locked decision from CONTEXT.md)

**Step 3a: Manual Mode**
- If user chooses Manual:
  - Show the ralph-tui launch command for their beads directory
  - Inform: "Run ralph-tui to execute beads interactively. When finished, signal back here with your results."
  - Wait for user signal (they will respond when done)
  - After user signals, scan .claude/pipeline/bead-results/ for any result files the user may have created
  - If result files exist, aggregate them. If not, ask user for pass/fail per bead or accept "all passed"
  - Write completion file and return

**Step 3b: Headless Mode**
- If user chooses Headless:
  - Record pre-execution git commit: PRE_EXEC_COMMIT=$(git rev-parse HEAD)
  - Store this in .claude/pipeline/pre-exec-commit.txt for review phase use
  - Create results directory: mkdir -p .claude/pipeline/bead-results
  - Build ordered list of beads from Glob results

**Step 4: Sequential Bead Execution (headless only)**
- For each bead in the ordered list, execute sequentially:
  1. Report progress: "Executing bead {i}/{total}: {BEAD_NAME}"
  2. Read the bead file content
  3. Execute via Bash:
     ```
     env -u CLAUDECODE claude -p "$(cat .beads/{bead}.md)" \
       --allowedTools "Read,Edit,Bash,Grep,Glob,Write" \
       --output-format json \
       --dangerously-skip-permissions
     ```
     CRITICAL: Must use `env -u CLAUDECODE` to avoid nested session error (Pitfall 3 from research).
     For large beads, use stdin piping: `cat .beads/{bead}.md | env -u CLAUDECODE claude -p --allowedTools "..." --output-format json --dangerously-skip-permissions`
  4. Capture exit code. Exit 0 = passed, non-zero = failed.
  5. Write result file to .claude/pipeline/bead-results/{BEAD_NAME}.md:
     ```yaml
     ---
     status: passed|failed
     bead: {BEAD_NAME}
     executed: {ISO8601 timestamp}
     ---
     ```
  6. Report result: "  -> {PASSED|FAILED}"
  7. If FAILED: STOP the batch immediately (locked decision). Report:
     "Batch stopped. {passed}/{total} beads passed, 1 failed."
     "Failed bead: {BEAD_NAME}"
     Do NOT continue to remaining beads.

**Step 5: Aggregate Results**
- After batch completes (all passed or stopped on failure):
  - Count: passed, failed, blocked from result files
  - Report summary: "{passed} passed, {failed} failed, {blocked} blocked out of {total} beads"

**Step 6: Handle Failure**
- If any bead failed, present failure gate:
  1. **re-run bead X** -- re-execute only the failed bead (targeted re-execution, locked decision)
  2. **retry all** -- re-run entire batch from start (clears all result files first)
  3. **proceed** -- accept partial results and continue to review phase
  4. **abort** -- stop pipeline
- If re-run bead X: re-execute that single bead, update its result file, then re-aggregate

**Step 7: Write Completion File**
- Write .planning/pipeline/execute.md with frontmatter:
  ```yaml
  completed: true
  mode: {headless|manual}
  total_beads: {N}
  passed: {count}
  failed: {count}
  blocked: {count}
  pre_exec_commit: {commit hash, for review phase scoping}
  ```
- Body: results table with per-bead status

**Step 8: Return Completion**
- Return: ## PHASE COMPLETE (if all passed or user chose proceed)
- Return: ## PHASE FAILED (if user chose abort)

CRITICAL implementation notes from research:
- MUST use `env -u CLAUDECODE` or `unset CLAUDECODE &&` before claude -p (Pitfall 3)
- MUST mkdir -p .claude/pipeline/bead-results before writing results (Pitfall 5)
- MUST quote all bead paths: basename "$bead" .md (Pitfall 7)
- No per-bead timeout (locked decision) -- trust claude -p to complete or fail
- No external quality check (locked decision) -- trust bead agent's self-reported result
- Show bead-level progress (locked decision)
- Result files are status-only (locked decision) -- no summary or details in result files
