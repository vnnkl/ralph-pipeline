<task type="auto">
  <name>Task 2: Write prd.md template with skill chaining, validation, and open questions extraction</name>
  <files>templates/prd.md</files>
  <action>
Replace the stub template content in templates/prd.md with a full subagent prompt. Keep the existing skeleton structure and {{VAR}} placeholders.

The instructions section must implement this flow:

**Step 1: Read Context**
- Read STATE.md and config.json from files_to_read
- Read depth setting from config.json via: `node ralph-tools.cjs config-get depth`
- If depth is "comprehensive": read ALL .planning/research/*.md files
- If depth is "standard" or "quick" (default): read only .planning/research/SUMMARY.md
- Read .planning/pipeline/clarify.md for scope and stack context

**Step 2: Invoke /ralph-tui-prd**
- Use the Skill tool to invoke /ralph-tui-prd
- Pass research context as the skill argument/context (the research content read in Step 1)
- The skill will conduct its own clarifying questions flow and produce a PRD
- NOTE: This template IS the direct dispatch subagent, so Skill tool IS available

**Step 3: Validate PRD Output**
After /ralph-tui-prd completes, validate the output:

3a. Check [PRD]...[/PRD] markers:
  - Search the output for [PRD] and [/PRD] markers
  - If missing: FAIL -- return "## PHASE FAILED\n\nPRD markers not found. /ralph-tui-prd may have failed or produced malformed output."

3b. Count user stories:
  - Search for ### US-NNN pattern within the [PRD] block
  - If fewer than 3: FAIL -- return "## PHASE FAILED\n\nOnly {N} user stories found. Minimum is 3 for a valid PRD."

3c. Validate tracer bullet US-001:
  - Parse the PRD for declared scope/layers (from Technical Considerations, Overview, or Architecture section)
  - Extract the layers the PRD declares as in-scope (e.g., "backend", "frontend", "database", "API", "CLI")
  - Parse US-001 description and acceptance criteria for references to each declared layer
  - If US-001 doesn't reference ALL declared layers: FAIL -- return "## PHASE FAILED\n\nUS-001 must be a vertical slice covering all declared layers: {layers}. Found coverage: {covered}. Missing: {missing}."
  - NOTE: Do NOT hardcode layers to DB+backend+frontend. Validate against what the PRD declares.

3d. Validate story structure:
  - Each US-NNN must have both Description and Acceptance Criteria sections
  - No duplicate US-NNN IDs

**Step 4: Extract Open Questions**
- Find the "## Open Questions" section in the PRD output
- Write each question to .planning/pipeline/open-questions.md as unchecked items:
  ```
  ---
  resolved: false
  count: {N}
  ---
  # Open Questions

  - [ ] {question text} (source: PRD)
  - [ ] {question text} (source: PRD)
  ```
- If no open questions section found, write the file with count: 0 and a note "No open questions from PRD"

**Step 5: Write PRD Output**
- Extract the content between [PRD] and [/PRD] markers
- Write to .planning/pipeline/prd.md with frontmatter:
  ```
  ---
  completed: true
  story_count: {N}
  tracer_bullet_valid: true
  open_questions: {count}
  ---
  ```
- Body: the full PRD content (between markers)
- Return: ## PHASE COMPLETE

Update success_criteria:

- PRD contains [PRD]...[/PRD] markers and >= 3 user stories
- US-001 covers all PRD-declared layers
- .planning/pipeline/open-questions.md exists with extracted questions

CRITICAL CONSTRAINTS (from CONTEXT.md locked decisions):

- Hard gate: markers missing or < 3 stories = phase FAILS visibly
- Tracer bullet validates against PRD-declared scope, NOT hardcoded layers
- Research context is configurable by depth setting (SUMMARY.md default, all files for comprehensive)
