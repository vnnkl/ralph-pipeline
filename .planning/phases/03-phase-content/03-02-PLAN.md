  <action>
Replace the stub template content in templates/resolve.md with a full subagent prompt. Keep the skeleton structure and {{VAR}} placeholders.

The instructions section must implement this flow:

**Step 1: Read Context**
- Read STATE.md and config.json from files_to_read
- Read .planning/pipeline/prd.md for the PRD content
- Read .planning/pipeline/open-questions.md for extracted questions from PRD phase

**Step 2: Scan for Unresolved Items**

2a. PRD Body Scan:
  Use Grep tool to search .planning/pipeline/prd.md for patterns:
  `\[TBD\]|\[TODO\]|\[PLACEHOLDER\]|TBD:|TODO:`

2b. Open Questions Scan:
  Read .planning/pipeline/open-questions.md
  Find all lines matching: `- [ ]` (unchecked checkbox items)

2c. Merge into ordered list of unresolved items. Each item has:
  - source: "PRD" or "open-questions"
  - text: the question or placeholder context
  - location: line number or section in PRD

**Step 3: Resolve Each Item One-by-One**
For each unresolved item:

3a. Read surrounding PRD context (5 lines before and after the marker)

3b. Generate 2-3 concrete answer options based on:
  - The surrounding PRD context
  - The project scope from clarify.md
  - Technical feasibility given the declared stack
  - Always include a "Let me explain my own answer" option as the last choice

3c. Present via AskUserQuestion:
  - Header: short label, max 12 characters (e.g., "DB Choice", "Auth", "API Style")
  - Question: the unresolved item with its surrounding context
  - Options: the 2-3 generated answers plus "Let me explain"

3d. Write answer inline to PRD immediately:
  - Replace the [TBD]/[TODO]/[PLACEHOLDER] marker with the user's chosen answer text
  - Write the updated PRD back to .planning/pipeline/prd.md
  - This ensures partial progress survives interruption

3e. If the item came from open-questions.md, mark it as resolved:
  - Change `- [ ]` to `- [x]` for that item
  - Write updated open-questions.md

3f. Vague answer detection:
  - If user's answer contains "TBD", "not sure", "maybe", "decide later", or similar vague language
  - Flag it: "This answer still contains a placeholder pattern. Please provide a concrete answer or explicitly mark as DECISION_PENDING: {fallback strategy}."
  - Re-ask the question

**Step 4: Final Validation Re-scan**
After all items are resolved:
- Re-scan PRD for [TBD], [TODO], [PLACEHOLDER], TBD:, TODO: patterns
- Re-scan open-questions.md for unchecked `- [ ]` items
- If ANY unresolved items remain: loop back to Step 2 (new patterns may have been introduced during revision)
- If clean: proceed to completion

**Step 5: Write Completion File**
- Update .planning/pipeline/open-questions.md frontmatter:
  ```
  ---
  resolved: true
  count: {total}
  resolved_count: {resolved}
  ---
  ```
- Write .planning/pipeline/resolve.md with frontmatter:
  ```
  ---
  completed: true
  items_resolved: {count}
  scan_passes: {count}
  ---
  ```
- Body: summary of resolved items and their answers
- Return: ## PHASE COMPLETE

- Write answers back inline to PRD immediately (partial progress survives interruption)
