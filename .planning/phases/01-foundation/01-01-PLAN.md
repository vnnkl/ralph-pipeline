  <name>Task 1: Create lib/core.cjs and lib/frontmatter.cjs</name>
  <files>lib/core.cjs, lib/frontmatter.cjs</files>
  <action>
Create lib/ directory. Build two modules modeled directly on GSD's equivalents.

**lib/core.cjs** — Shared helpers used by every other module:
- `output(result, raw, rawValue)`: JSON to stdout. If raw mode and rawValue provided, write raw string. Otherwise JSON.stringify with 2-space indent. If JSON > 50000 chars, write to temp file and output `@file:{path}`. Exit 0.
- `error(message, code)`: JSON error object `{ error: true, message, code }` to stderr. Exit 1. Code defaults to 'UNKNOWN'. This extends GSD's pattern per CONTEXT.md decision on error codes.
- `safeReadFile(filePath)`: fs.readFileSync wrapped in try/catch, returns null on failure.
- `execGit(cwd, args)`: child_process.execSync for git commands. Returns `{ stdout, exitCode }`. Catches errors and returns exitCode != 0 on failure.
- `pathExistsInternal(cwd, relativePath)`: path.join + fs.existsSync.
- `loadConfig(cwd)`: Load `.planning/config.json` with defaults: `{ mode: 'normal', depth: 'standard', model_profile: 'balanced', commit_docs: true, auto_advance: false, time_budget: null, ide: null }`. Merge file values over defaults using spread. Return defaults if file missing/invalid.
- `saveConfig(cwd, config)`: Write config object to `.planning/config.json` with JSON.stringify 2-space indent.
- Export all functions via module.exports.

**lib/frontmatter.cjs** — YAML frontmatter parser modeled on GSD's `~/.claude/get-shit-done/bin/lib/frontmatter.cjs`:
- `extractFrontmatter(content)`: Detect `---` delimiters, parse key-value pairs from YAML frontmatter. Return `{ frontmatter: {}, body: string, hasFrontmatter: boolean }`. Handle nested arrays/objects via simple YAML subset parsing (strings, numbers, booleans, arrays with `- item` syntax, null). Do NOT use an npm YAML parser.
- `reconstructFrontmatter(frontmatter, body)`: Rebuild the full file content from a frontmatter object and body string. Serialize object back to YAML subset format with `---` delimiters.
- `spliceFrontmatter(content, updates)`: Extract frontmatter, merge updates into it (immutable — spread existing + updates), reconstruct. Returns full file content string.
- Export all functions via module.exports.

Use only Node.js builtins: fs, path, child_process, os. No npm dependencies.
