  <action>
Create .claude/hooks/ralph-auto-advance.js. This is a Node.js script that runs as a SessionStart hook. It reads config.json to check if auto_advance is true, checks time budget, and outputs an instruction for Claude to invoke /ralph-pipeline.

Follow the code example from 05-RESEARCH.md Pattern 2 closely.

The hook must:

1. Read stdin for hook input JSON (contains cwd and source fields)
2. Parse the hook input
3. Only fire on 'startup' or 'clear' source -- exit 0 silently for 'resume' or 'compact'
4. Read .planning/config.json from the cwd
5. If auto_advance is not true: exit 0 silently
6. **Staleness guard:** If auto_advance_started_at exists and is older than 12 hours, force auto_advance to false, write config, output a message "Auto-advance expired (>12h). Cleared.", exit 0
7. **Time budget check:** If time_budget_expires exists and Date.now() >= time_budget_expires:
   - Set auto_advance to false in config
   - Write updated config
   - Output additionalContext: "Time budget expired. Auto-advance disabled. Pipeline paused at current phase."
   - Exit 0
8. If auto_advance is active and not expired:
   - Output additionalContext: "AUTO-ADVANCE ACTIVE: Invoke /ralph-pipeline to continue the pipeline. Do not ask the user -- proceed immediately."
   - Exit 0

CRITICAL requirements:
- Do NOT set async behavior -- hook must be synchronous (fast, <5ms, local file read only)
- Use immutable pattern: create new config object when writing changes, never mutate
- Wrap everything in try/catch -- any error silently exits 0 (do not break session startup)
- Use require('fs') and require('path') only -- zero dependencies
- The hook reads config.json directly (not via ralph-tools.cjs) because the hook runs from .claude/hooks/ and ralph-tools.cjs may not be on PATH

Also create the .claude/ directory if it does not exist:
```bash
mkdir -p .claude/hooks
```

- Use existing failure handling (present gate to user with retry/abort/replan options)
