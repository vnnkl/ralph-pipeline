From lib/phase.cjs -- cmdPhaseComplete(cwd, phaseNumber, raw):
  Called via CLI: `node ralph-tools.cjs phase-complete <dev-phase-number>`
  Returns JSON: { completed: N, next_phase: M, date: "YYYY-MM-DD" }

PIPELINE_PHASES array (ids 1-9):

**Why the ROADMAP guard:** `cmdPhaseComplete` always advances STATE.md Phase field, even if ROADMAP is already checked. Without this guard, resuming after a completed pipeline would double-advance the dev-phase. The grep check is the simplest guard -- it detects whether ROADMAP already has `[x]` for the current dev-phase. If so, phase-complete already ran successfully and should not be called again.

**Key constraint:** The auto-correct NEVER writes pipeline sub-phase info into the STATE.md Phase field. Per CONTEXT.md locked decision, Phase field stays at dev-phase level. Only the Status field gets pipeline sub-phase tracking.
