**Structure the skill file with these steps (mirroring SKILL.md but scoped to phases 1-7):**

**Step 0: Resolve Paths** -- identical to SKILL.md Step 0. Extract RALPH_TOOLS and PROJECT_CWD.

**Step 1: Load State** -- identical to SKILL.md Step 1. Run `init pipeline`, parse JSON.

**Step 1b: Setup Wizard** -- NEW (replaces SKILL.md's time budget prompt). Only runs if `config.marathon` is not `true` (first marathon run or fresh start):

Three questions presented sequentially via AskUserQuestion:

Question 1 - Time Budget:
- Header: "Time Budget"
- Question: "How many hours for execution? (Planning has no time limit)"
- Options: 1. 2 hours / 2. 4 hours / 3. 8 hours / 4. No limit / 5. Custom
- Store: `config-set marathon_time_budget_hours {hours}` (or skip if "No limit")
- CRITICAL: Do NOT call `time-budget start` -- this happens when execute phase begins (outside marathon scope)

Question 2 - Bead Format:
- Header: "Bead Format"
- Question: "Which bead output format?"
- Options: 1. bd -- Go beads / 2. br -- Rust beads / 3. prd.json -- JSON format
- Store: `config-set bead_format {format}`

Question 3 - YOLO Mode:
- Header: "Mode"
- Question: "Auto-approve all planning gates?"
- Options: 1. Yes -- YOLO mode (fully unattended planning) / 2. No -- Interactive (pause at each gate)
- Store: `config-set mode {yolo|normal}`

**YOLO skip rule:** If mode is already "yolo" when marathon is invoked, skip the wizard entirely. Use existing config values. Only prompt for `bead_format` if not already set.

After wizard (or skip), set marathon flag:
```bash
node {RALPH_TOOLS} --cwd {PROJECT_CWD} config-set marathon true
```

**Step 2: Position Detection** -- Use scan-phases to find first incomplete phase among phases 1-7. Marathon is resumable: if phases 1-3 are complete, resume from phase 4.

```bash
node {RALPH_TOOLS} --cwd {PROJECT_CWD} scan-phases
```

Parse JSON array. Filter to phases 1-7 only (ids 1 through 7). Find first where `completed` is false. If all 7 are complete, skip directly to the bead review gate (Step 8 in Plan 15-02).

Auto-skip completed phases with a log line each.
