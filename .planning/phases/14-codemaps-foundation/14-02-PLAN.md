**Template structure:**

```markdown
<objective>
Generate codebase context maps by spawning 4 parallel mapper agents.
Each agent analyzes the project and writes concise summaries to .planning/codebase/.
</objective>

<instructions>
## Codemap Generation

You are a codemap orchestrator. Your job is to ensure .planning/codebase/ contains fresh, accurate codebase summaries by spawning 4 parallel mapper agents.

### Step 1: Prepare Directory

```bash
mkdir -p {{CWD}}/.planning/codebase
```

Remove stale files from prior inventory (old names that are no longer in the decided set):

```bash
rm -f {{CWD}}/.planning/codebase/INTEGRATIONS.md {{CWD}}/.planning/codebase/TESTING.md
```

### Step 2: Spawn 4 Parallel Mapper Agents

[4 mapper agent prompts -- see below]

### Step 3: Wait and Verify

[wait + file checks]

### Step 4: Return Completion

Return: ## CODEMAP GENERATION COMPLETE

</instructions>
```

**Mapper Agent Prompts (inlined -- no Skill tool, per CMAP-08):**

Each mapper is spawned as a Task with `run_in_background=true`. Each prompt:
1. States the agent's role clearly
2. Lists the exact files it must write (with absolute paths using {{CWD}})
3. Specifies the analysis focus areas
4. Sets the target length (200-400 lines per file)
5. Uses heredoc write pattern for .planning/ files (Bash tool, not Write tool)

**Agent 1 -- Stack & Dependencies Mapper:**
Files: STACK.md, DEPENDENCIES.md
Focus:
- STACK.md: Languages, runtime versions, frameworks, build tools, package managers
- DEPENDENCIES.md: npm/pip/cargo dependencies, external services, integrations, version constraints

**Agent 2 -- Architecture & API Mapper:**
Files: ARCHITECTURE.md, API.md
Focus:
- ARCHITECTURE.md: System layers, data flow, component boundaries, design patterns, state management
- API.md: Public API surface, CLI commands, exported functions, route handlers, configuration interfaces

**Agent 3 -- Structure & Conventions Mapper:**
Files: STRUCTURE.md, CONVENTIONS.md
Focus:
- STRUCTURE.md: File tree with directory purposes, module organization, entry points, build outputs
- CONVENTIONS.md: Naming patterns, code style, error handling patterns, testing patterns, file organization rules

**Agent 4 -- Concerns Mapper:**
Files: CONCERNS.md
Focus:
- CONCERNS.md: Tech debt, fragile code, missing test coverage, security gaps, performance bottlenecks, known bugs, areas needing refactoring

**Each mapper prompt must include:**
- "You are a codebase mapper agent." preamble
- Explicit file paths: `{{CWD}}/.planning/codebase/FILENAME.md`
- "Write each file with 200-400 lines max. Focus on signal, not noise."
- "Use the Bash tool with heredoc to write files under .planning/"
- "Read the project source code at {{CWD}} to gather information. Use Glob, Grep, and Read tools to analyze the codebase."
- "Do NOT use the Skill tool."

**Step 3 (Wait and Verify):**
After all 4 agents complete, verify all 7 files exist:
```bash
ls -la {{CWD}}/.planning/codebase/STACK.md {{CWD}}/.planning/codebase/ARCHITECTURE.md {{CWD}}/.planning/codebase/STRUCTURE.md {{CWD}}/.planning/codebase/CONCERNS.md {{CWD}}/.planning/codebase/CONVENTIONS.md {{CWD}}/.planning/codebase/DEPENDENCIES.md {{CWD}}/.planning/codebase/API.md
```

If any file is missing, log a warning: "Mapper agent failed to produce {filename}. Proceeding with available files." Do NOT fail the entire generation because one agent failed (per CONTEXT.md -- Claude's discretion for error handling).

**IMPORTANT constraints:**
- Do NOT include `{{PHASE_FILES}}` or `{{CODEMAP_FILES}}` placeholders in this template -- it does not receive upstream phase files
- DO include `{{CWD}}` and `{{RALPH_TOOLS}}` -- these are filled by the orchestrator
- The template does NOT call `ralph-tools.cjs codemap check` -- freshness checking is the orchestrator's responsibility (SKILL.md), not the template's
- No `<files_to_read>` block -- mapper agents discover the codebase themselves

**File writing instructions for mapper agents:**

```"
