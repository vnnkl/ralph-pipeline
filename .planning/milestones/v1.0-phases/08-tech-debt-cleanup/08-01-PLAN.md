**Objective section:** Execute pre-flight phase (phase 1 of 9). Run pre-flight environment checks, display results, handle setup actions interactively, and block pipeline if required dependencies are missing.

**files_to_read section:** Include {{STATE_PATH}}, {{CONFIG_PATH}}, and {{PHASE_FILES}} template variables (same as other templates). Do NOT include ralph-tools.cjs -- that was a stub artifact.

**Instructions section:**

Step 1: Read Context -- Read files listed in files_to_read.

Step 1.5: Check YOLO Mode -- Run `node ralph-tools.cjs config-get mode --raw`. Set YOLO_MODE flag.

Step 2: Run Preflight Checks -- Run `node ralph-tools.cjs preflight --raw`. Capture stdout and exit code.

On exit code 0 (passed):
- Parse JSON from stdout
- Display checklist summary: skills N/N found, MCP servers N/N found, GSD reference version+status, configured IDE
- Show optional missing items (required: false) as non-blocking warnings with install hints
- Proceed to Step 3

On exit code 1 (failed):
- Parse stdout (NOT stderr) for the full result JSON
- Display each missing required item (required: true) with type, name, purpose, install_hint
- If YOLO_MODE: Log "YOLO mode cannot bypass missing required dependencies" then return ## PHASE FAILED
- If not YOLO_MODE: Present via AskUserQuestion with Header "Pre-flight", options: "Install and retry" / "Retry" / "Abort pipeline"
  - Options 1 or 2: re-run preflight (max 3 retries)
  - Option 3: return ## PHASE FAILED

Step 3: Handle Setup Actions -- From preflight result, iterate setup_actions array:
- add_gitignore: Run `node ralph-tools.cjs setup-gitignore {pattern}`
- init_planning: Create missing planning files via `mkdir -p .planning` and minimal file creation
- ask_ide: If YOLO_MODE, default to "vscode" via `node ralph-tools.cjs config-set ide vscode`. If not YOLO_MODE, present IDE options via AskUserQuestion (Header: "IDE", options from the action's options array), then run `node ralph-tools.cjs config-set ide {choice}`

Step 4: Write Completion File -- Write .planning/pipeline/preflight.md with frontmatter (completed: true, passed: true, missing_optional list, setup_actions_taken list, ide value) and body with summary.

Step 5: Return Completion -- Return `## PHASE COMPLETE`
