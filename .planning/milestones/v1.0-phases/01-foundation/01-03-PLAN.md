  <name>Task 1: Create lib/preflight.cjs — dependency checks and GSD reference setup</name>
  <files>lib/preflight.cjs</files>
  <action>
Build the pre-flight module. This is ralph-specific (GSD has no equivalent). Model the output pattern on GSD's JSON conventions.

**cmdPreflight(cwd, raw):**

Collect ALL missing dependencies into an array, then return structured result. Never exit early on first missing item — collect everything so user fixes all at once.

**Check categories (in order):**

1. **Required skills** — Check SKILL.md exists on disk:
   ```javascript
   const requiredSkills = [
     { name: 'ralph-tui-prd', purpose: 'PRD creation' },
     { name: 'ralph-tui-create-beads', purpose: 'Bead conversion (Go)' },
   ];
   ```
   Search paths (in order): `~/.claude/skills/{name}/SKILL.md`, then `{cwd}/.claude/skills/{name}/SKILL.md`. Use os.homedir() for ~.
   If not found in either location, add to missing: `{ type: 'skill', name, purpose, install_hint: 'Install via Claude Code skill system' }`.

2. **Required MCP servers** — Check if configured MCP servers are reachable:
   ```javascript
   const requiredMcpServers = [
     { name: 'context7', purpose: 'Documentation lookups', config_key: 'context7' },
   ];
   ```
   Check by reading Claude's MCP config file (`~/.claude.json` or `~/.claude/settings.json`) and verifying the server name exists in the `mcpServers` config. Use `safeReadFile` to read the config, JSON.parse, and check for the key.
   If not found in MCP config, add to missing: `{ type: 'mcp_server', name, purpose, required: true, install_hint: 'Add MCP server via Claude Code settings or .claude.json' }`.
   Note: This is a config-level check (is the server configured?), not a runtime connectivity check. The list of required servers is a constant that can be extended as the pipeline grows.

3. **Optional CLIs** — Check if bd/br exist on PATH:
   ```javascript
   const optionalCLIs = [
     { name: 'bd', purpose: 'Go bead CLI', required: false },
     { name: 'br', purpose: 'Rust bead CLI', required: false },
   ];
   ```
   Check via `child_process.execSync('which {name}')` (wrapped in try/catch — catch means not found). On Windows, use `where` instead of `which`.
   If not found, add to missing but mark `required: false` — these are optional depending on bead format choice.

4. **GSD reference** — Check `.reference/get-shit-done/` exists in cwd:
   If missing, attempt to set up:
   a. Check if `~/.claude/get-shit-done/` exists (the plugin-installed copy).
   b. If yes, record `{ type: 'reference', name: 'get-shit-done', source: 'local', local_path: homeGsdPath }`. The orchestrator will handle the actual copy (preflight just reports what's available).
   c. If no local copy, record `{ type: 'reference', name: 'get-shit-done', source: 'none', install_hint: 'GSD must be installed via Claude Code plugin system first' }`.
   d. Read VERSION from the source if available, include as `version` field.

5. **.gitignore check** — Verify `.reference/` is in .gitignore:
   Read .gitignore, check if `.reference/` or `.reference` pattern exists. If not, add to `setup_actions`: `{ action: 'add_gitignore', pattern: '.reference/' }`.

6. **.planning/ directory** — Check if `.planning/` exists with required files (STATE.md, ROADMAP.md, PROJECT.md):
   If missing, add to `setup_actions`: `{ action: 'init_planning', missing_files: [...] }`.

7. **IDE preference** — Load config, check if `ide` is set:
   If null, add to `setup_actions`: `{ action: 'ask_ide', options: ['vscode', 'cursor', 'zed', 'neovim', 'other'] }`.

**Return format:**
```javascript
output({
  passed: missing.filter(m => m.required !== false).length === 0 && referenceOk,
  missing: missing,           // Array of missing dependencies
  setup_actions: actions,     // Array of setup tasks for orchestrator
  reference: referenceInfo,   // GSD reference status object
  config: { ide: config.ide },
}, raw);
```

If `passed` is false AND there are required missing items, write a summary to stderr AND exit 1 — this is a blocking error, not a warning. The orchestrator also reads `passed: false` from the JSON, but the non-zero exit code makes it unambiguous to any caller. Exit 0 only when all required checks pass (optional missing items like bd/br do not block).

**Important:** Do NOT actually perform installations or copies. Preflight is diagnostic only — it reports what's missing and what actions are available. The orchestrator handles the interactive installation flow (Phase 2 scope).

**Actionable install hints:** Every missing item MUST include an `install_hint` string with a concrete, actionable command or step the user can take. Examples:
- Skills: `'Install via: claude install-skill ralph-tui-prd'`
- MCP servers: `'Add to ~/.claude.json mcpServers config or run: claude mcp add context7'`
- CLIs: `'Install bd via: go install github.com/...'` or `'Install br via: cargo install ...'`
- GSD reference: `'Install GSD plugin: claude install get-shit-done'`

This ensures that even without Phase 2's interactive installer, preflight output is actionable. Users can copy-paste the hint to fix each issue. Phase 2 will wrap these hints in an interactive prompt flow.
