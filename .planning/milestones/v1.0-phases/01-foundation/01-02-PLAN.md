**GREEN phase — Implement lib/state.cjs:**

Model on GSD's `~/.claude/get-shit-done/bin/lib/state.cjs`. Key functions:

- `stateExtractField(content, fieldName)`: Regex `**{fieldName}:**\s*(.+)` (case-insensitive). Return match[1].trim() or null.
- `stateReplaceField(content, fieldName, newValue)`: Regex replace `(**{fieldName}:**\s*)(.*)` with `$1{newValue}`. Return new string or null if field not found. IMMUTABLE — do not mutate input.
- `buildStateFrontmatter(bodyContent, cwd)`: Extract Phase, Status, Last activity, Plan fields from body. Build object: `{ current_phase, status, last_activity, current_plan }`. Parse phase number from "N of M" pattern.
- `syncStateFrontmatter(content, cwd)`: Extract existing frontmatter, build new from body, merge (body wins), reconstruct.
- `writeStateMd(statePath, content, cwd)`: Call syncStateFrontmatter, then fs.writeFileSync.
- `cmdState(cwd, subcommand, args, raw)`: CLI handler for `state` command. Subcommands: `get {field}` (extract and output), `set {field} {value}` (replace and write), `json` (output frontmatter as JSON).

Run tests — they MUST pass.

**lib/phase.cjs** — Phase lifecycle modeled on GSD's `~/.claude/get-shit-done/bin/lib/phase.cjs`:

- `findPhaseInternal(cwd, phaseNumber)`: Look up phase directory in `.planning/phases/`. Phase directories named `NN-name` (e.g., `01-foundation`). Pad phaseNumber to 2 digits. Find matching directory. Return `{ number, name, directory, path }` or null.
- `cmdPhaseComplete(cwd, phaseNumber, raw)`: Validates phase exists via findPhaseInternal. Then:
  1. Read ROADMAP.md, mark phase checkbox `[x]`, update progress table status to "Complete", add date
  2. Read STATE.md, advance current_phase to next phase number using stateReplaceField, update Status to next phase name, update Last activity date
  3. Write both files via writeStateMd (for STATE.md) and fs.writeFileSync (for ROADMAP.md)
  4. Output `{ completed: phaseNumber, next_phase: nextNumber, date: isoDate }`
  5. If phase not found: error('Phase not found', 'PHASE_NOT_FOUND')
  6. If already last phase: output `{ completed: phaseNumber, next_phase: null, pipeline_complete: true }`

- Export findPhaseInternal, cmdPhaseComplete.

**lib/commands.cjs** — Git commit with conditional logic, modeled on GSD's commands.cjs:

- `cmdCommit(cwd, message, files, raw)`:
  1. Load config. If `commit_docs` is false, output `{ committed: false, reason: 'skipped_commit_docs_false' }` and return.
  2. Check if .planning is gitignored: `execGit(cwd, ['check-ignore', '-q', '.planning'])`. If exit 0 (ignored), output `{ committed: false, reason: 'skipped_gitignored' }` and return.
  3. Stage files: if files array provided and non-empty, `git add` each file. Otherwise `git add .planning/`.
  4. Commit with message. If nothing to commit (exit != 0), output `{ committed: false, reason: 'nothing_to_commit' }`.
  5. On success, get short hash via `git rev-parse --short HEAD`, output `{ committed: true, hash }`.
- Export cmdCommit.
