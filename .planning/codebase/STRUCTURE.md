- Purpose: Custom hooks registered during pre-flight for compaction resilience
- Generated: Yes (`settings.json` updated during pre-flight)
- Committed: No (project-local)
- Hook type: SessionStart hook that re-injects pipeline state after context compaction

## Pattern: How the Orchestrator Works

```
1. User invokes /ralph-pipeline
2. Main agent (thin)
   ├─ Pre-flight: Check deps, init git, create .claude/pipeline/, init .ralph-tui/config.toml, register hook
   ├─ Read .claude/pipeline/state.md (or detect phase from filesystem)
   ├─ Phase 0: Dispatch Task subagent for codemap (reads phase-0-orient.md output)
   ├─ Phase 1: Run interactive rounds in main agent (AskUserQuestion x3)
   ├─ Phase 2: Dispatch Task subagent for research (reads phase-2-research.md output)
   ├─ Phase 3: Dispatch Task subagent for PRD (reads phase-3-prd.md output)
   ├─ Phase 4: Dispatch Task subagent for deepening (reads phase-4-deepen.md output)
   ├─ Phase 4.5: Run interactive question resolution in main agent (AskUserQuestion for each open question)
   ├─ Phase 5: Dispatch Task subagent for conversion (reads phase-5-convert.md output, validates)
   ├─ Phase 6: Instruction-only (user runs rally-tui or headless via script)
   ├─ Phase 7: Dispatch Task subagent for review (reads phase-7-review.md output)
   ├─ Phase 8: Dispatch Task subagent for harvest (reads phase-8-harvest.md output)
   └─ Each gate: AskUserQuestion to decide next action (proceed, refine, skip)
```

## File Read/Write Patterns

**Main agent reads:**
- `state.md` (every phase transition)
- `phase-[N]-[name].md` (to verify `completed: true`)
- `open-questions.md` (for Phase 4.5)

**Main agent writes:**
- `state.md` (updates `current_phase`, `project_type`, `tech_stack`, `quality_gates`, `research_agents`, `review_agents`, `convert_format`, `open_questions_resolved`)
- Files in `.claude/pipeline/` (via subagent output verification)

**Subagents read:**
- Input files specified in Task prompt (e.g., `state.md`, `phase-[N-1]-name.md`, `phase-0-orient.md`)

**Subagents write:**
- Phase output file (e.g., `phase-2-research.md`) with `completed: true/false`
- `open-questions.md` (append new questions if any arise)

## State Machine

```
state.md { current_phase: N }
   ↓
1. Read phase-N file
2. If phase-N.completed == false:
   → Offer to re-run phase N
3. If phase-N.completed == true:
   → Run gate (AskUserQuestion)
   → Update state.md { current_phase: N+1 }
   → Proceed to phase N+1
4. After compaction:
   → SessionStart hook injects state.md
   → Resume at current phase
```
